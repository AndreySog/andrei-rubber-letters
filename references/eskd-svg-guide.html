01

Общие положения

основа

Каждый чертёж — один автономный inline-SVG без внешнего CSS: все атрибуты заданы на элементах, поэтому файл одинаково отображается в README, на GitHub Pages и в любом браузере. Человек читает разделы по порядку; ИИ-агенту достаточно JSON-констант ниже, промпта из §11 и чек-листа §12.

1.1
Масштаб экрана: 1 мм = 3 px. Все координаты — размер в миллиметрах, умноженный на 3.
1.2
Один цвет туши #1A1A18 на белом фоне. Ни теней, ни градиентов, ни скруглений «для красоты».
1.3
fill="none" по умолчанию; заливка — только штриховка (pattern) и стрелки (marker).
1.4
Порядок слоёв снизу вверх: оси → невидимый контур → видимый контур → штриховка → резьба → размеры → текст → позиции.
1.5
Числа — с десятичной запятой («0,95»); единицы «мм» на чертеже не пишут. Координаты кратны 0,5 px.
1.6
Текст никогда не масштабируется transform-ом; допустим только rotate(-90 x y) для вертикальных размеров.
eskd.tokens.json — константы для системного промпта 
{
  "scale_px_per_mm": 3,
  "ink": "#1A1A18",
  "paper": "#FFFFFF",
  "stroke": { "толстая": 2.4, "тонкая": 1.0, "разомкнутая": 3.6 },
  "dash":   { "штриховая": "12 4", "осевая": "24 5 3 5" },
  "hatch":  { "угол": 45, "шаг_px": 9 },
  "arrow":  { "длина_px": 12, "ширина_px": 4.8 },
  "font":   { "семейство": "PT Sans", "стиль": "italic",
              "размерные_числа_px": 15, "заголовки_px": 21 },
  "слои": ["оси", "невидимый контур", "видимый контур", "штриховка",
           "резьба", "размеры", "текст", "позиции"]
}
ДЛЯ ИИ Скопируй JSON целиком в системный промпт. Любое значение толщины, цвета или шрифта вне этого JSON — ошибка построения.

2.1
На чертеже ровно два рабочих значения толщины: 2.4 (толстая) и 1 (все тонкие). Исключение — разомкнутая линия сечения: 3.6.
2.2
Осевые выходят за контур на 6–15 px, начинаются и заканчиваются длинным штрихом, в центре отверстия пересекаются штрихами.
2.3
Невидимый контур (штриховая) наносится только там, где без него чтение геометрии невозможно.
2.4
stroke-linecap не задавать (butt по умолчанию); соединения контуров — без скруглений.
ДЛЯ ИИ Валидация: собери множество всех stroke-width в готовом SVG — оно обязано быть подмножеством {2.4, 1, 3.6}.
03

Шрифт

ГОСТ 2.304-81
Размер 10 — АБВГДабвгд 0123456789
Размер 7 — заголовки видов, номера позиций
Размер 5 — размерные числа и надписи (основной)
⌀40   М20×1,5   R10   2,5×45°   ±0,1   l₀   D₁
3.1
Один шрифт на весь чертёж: 'PT Sans', курсив — веб-приближение наклонного шрифта типа Б (75°). Подключается с Google Fonts.
3.2
Высоты: размерные числа — 15 px (≈5 мм); индексы — 11 px; заголовки видов и номера позиций — 21 px; подписи «Рис. N» — 15–17 px.
3.3
Спецсимволы — юникодом: ⌀ (U+2300), ° (U+00B0), × (U+00D7), ± (U+00B1); индексы — ₀ ₁ ₂. Обозначение резьбы — кириллическая заглавная М.
3.4
Обозначения величин (d, l, S, D, h) — курсивом и на чертеже, и в таблицах, и в тексте задания.
04

Размеры

ГОСТ 2.307-68
4.1
Размерная линия — тонкая, параллельна измеряемому отрезку, на расстоянии ≥ 30 px от контура; между параллельными размерными ≥ 21 px.
4.2
Выносные — тонкие, перпендикулярны размерной; зазор от контура 0–3 px, выход за размерную 3–6 px.
4.3
Стрелки — заполненные, длина 12 px, ширина 4.8 px, одинаковые на всём чертеже (один marker). Если между выносными < 36 px — стрелки снаружи, остриями внутрь.
4.4
Число — над размерной линией с зазором 3–4 px, по центру. На вертикальных размерах — rotate(-90), читается справа.
4.5
Меньшие размеры — ближе к контуру, чтобы размерные и выносные линии не пересекались. Каждый размер ставится один раз.
4.6
Знаки: ⌀ перед диаметром; R перед радиусом (одна стрелка, в дугу); М — резьба (М20, М20×1,5); фаски — «2,5×45°».
dimension.svg — минимальный размер со стрелками 
{{ codeDim }}
ДЛЯ ИИ Все стрелки — один marker «ar» с orient="auto-start-reverse". Не рисуй стрелки полигонами вручную: они разъедутся по углу и размеру.
05

Штриховка и разрезы

ГОСТ 2.306-68
Втулка в продольном разрезе — обе зоны сечения штрихуются одинаково
отверстие между зонами не штрихуется
5.1
Металлы — тонкие линии под 45°, шаг 9 px (допустимо 6–15). Реализация — SVG-паттерн с шагом, зафиксированным в координатах листа (сниппет ниже).
5.2
В пределах одной детали угол и шаг одинаковы на всех зонах сечения (общий pattern это гарантирует).
5.3
Смежные детали — встречный наклон (135°) или другой шаг (6 px).
5.4
Крепёж, валы, оси и рёбра жёсткости в продольном разрезе не штрихуют.
5.5
Тонкие стенки (< 6 px) — сплошная заливка. Размерное число внутри штриховки — в «окне» (прямоугольник fill="#FFFFFF" под текстом).
hatch.svg — паттерн штриховки 
{{ codeHatch }}
ДЛЯ ИИ Один pattern на деталь. Для смежной детали объяви второй pattern (id="ht2", rotate(135)) — не меняй существующий.
06

Компоновка листа

ГОСТ 2.301 · 2.104 · 2.305
6.1
Форматы: А4 — 210×297 мм, А3 — 297×420 мм; в px — ×3. Лист всегда с рамкой: поля слева 20 мм (60 px), остальные 5 мм (15 px), линия рамки — толстая.
6.2
Основная надпись 185×55 мм — вплотную к рамке в правом нижнем углу.
6.3
Метод первого угла (ГОСТ 2.305): вид сверху — под главным, вид слева — справа от главного. Виды строго в проекционной связи: общие габариты выровнены по осям.
6.4
Учебная страница (как эталон §9): текст задания → таблица данных → чертёж с подписью «Рис. N» по центру.
07

Таблицы

ГОСТ 2.105
7.1
Внешняя рамка — 2.4 px, внутренние линии — 1 px, border-collapse:collapse. Шрифт — 'PT Sans' 15 px.
7.2
Надпись «Таблица N» — над таблицей справа; название — по центру, полужирным.
7.3
Числа — по центру ячеек; заголовки строк — слева с отступом 14 px; высота строки ≥ 30 px.
7.4
Обозначения величин в заголовках — курсивом (d, P, S, D, h), как на чертеже. Образец — Таблица 10 в §9.

08

Сборочные чертежи

ГОСТ 2.109
8.1
Каждой детали — номер позиции на полке-выноске: точка (marker) на детали → тонкая линия → горизонтальная полка 30–40 px → номер над полкой.
8.2
Шрифт позиций — 21 px, в 1,4 раза крупнее размерных чисел. Полки выравнивать в колонку или ряд; линии-выноски не пересекаются.
8.3
Болт (1) и гайка (2) в продольном разрезе не штрихуются; смежные пластины (3, 4) — встречная штриховка.
8.4
Спецификация — таблицей по правилам §7: Поз. / Обозначение / Наименование / Кол.
